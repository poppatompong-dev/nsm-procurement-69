# Project Overview (ภาพรวมโครงการ)
**ระบบตรวจสอบและรายงานพัสดุคอมพิวเตอร์ (49 รายการ) - เทศบาลนครนครสวรรค์**

## วัตถุประสงค์ (Objectives)
1. เพื่อใช้เป็นเครื่องมือตรวจรับพัสดุคอมพิวเตอร์ที่จัดซื้อรวมจำนวน 49 รายการ สำหรับกองยุทธศาสตร์และงบประมาณ เทศบาลนครนครสวรรค์
2. เพื่อช่วยให้การจับคู่รายการวัสดุกับหลักฐานรูปภาพถ่ายจริง (Photo Evidence) มีความถูกต้อง และค้นหาพัสดุที่ยังไม่มีรูปภาพประกอบได้ทันที
3. เพื่อรองรับการตรวจรับพัสดุนอกสถานที่ (สมาร์ตโฟน/แท็บเล็ต) และแชร์สถานะระหว่างคณะกรรมการได้สะดวกรวดเร็ว
4. เพื่อรองรับการพิมพ์รายงานแนบท้ายการตรวจรับในรูปแบบ PDF และส่งออกรายงานในรูปของไฟล์ Excel

## กลุ่มผู้ใช้งาน (Target Users)
* **คณะกรรมการตรวจรับพัสดุ:** ตามคำสั่งเทศบาลนครนครสวรรค์ ที่ ๘๖๔/๒๕๖๙ (ประธานและกรรมการรวม 3 ท่าน)
* **เจ้าหน้าที่พัสดุ/จัดซื้อ:** สำหรับการรวบรวมหลักฐานและตรวจสอบสถานะการส่งมอบพัสดุ
* **ผู้ตรวจสอบภายใน (Internal Auditor):** สำหรับการประเมินความโปร่งใสและตรวจสอบความครบถ้วนของพัสดุจัดซื้อ

## ลำดับการทำงานของระบบ (Business Flow)
1. **การโหลดข้อมูลพัสดุเริ่มต้น:** โหลดข้อมูลพัสดุ 49 รายการพร้อมข้อมูลจำเพาะ ราคา และรูปภาพที่จับคู่ไว้เริ่มต้น
2. **การคัดกรองฐานข้อมูล:** คณะกรรมการสามารถค้นหา คัดกรองตามกลุ่มงานผู้เบิก หรือแยกแยะพัสดุที่ยังไม่มีรูปภาพได้
3. **การตรวจสอบและอัปเดต:** คณะกรรมการตรวจสอบพัสดุจริงหน้างาน หากถูกต้องให้สลับสถานะเป็น "ผ่านการตรวจรับ" และบันทึกรูปภาพหรือเขียนโน้ตเพิ่มเติมได้
4. **การแชร์สถานะการตรวจรับ:** กดปุ่ม "แชร์สถานะให้ผู้อื่น" เพื่อคัดลอกลิงก์ที่บันทึกความคืบหน้าส่งให้ทีมงาน
5. **การออกรายงาน:** สลับไปแท็บออกรายงาน เพื่อพิมพ์รายงาน PDF (พร้อมระบบจัดรูปตารางและแนบชื่อกรรมการลงท้ายอัตโนมัติ) หรือดาวน์โหลดไฟล์ Excel เข้าเครื่อง
6. **การเริ่มโครงการตรวจรับรอบถัดไป:** ที่แท็บ "จัดการโครงการ" เจ้าหน้าที่สามารถโคลนโครงการเดิม (คัดลอกแม่แบบและคณะกรรมการอัตโนมัติ) แก้ไขเฉพาะชื่อโครงการ/เลขที่สัญญา/ผู้ขาย/ปีงบประมาณ แล้วนำเข้าใบเสนอราคา Excel ของรอบใหม่ได้ทันที โดยไม่กระทบข้อมูลของโครงการเดิม
