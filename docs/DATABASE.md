# Database Documentation (เอกสารฐานข้อมูลพัสดุ)

ระบบเก็บข้อมูลของพัสดุและคณะกรรมการในรูปแบบ JSON โครงสร้างแบบ Client-side Database และบีบอัดสถานะส่งผ่านทางบราวเซอร์

## 1. Procurement Data Structure (โครงสร้างไฟล์ procurementData.json)
ฐานข้อมูลพัสดุเริ่มต้นถูกจัดเก็บเป็น Array ของวัตถุที่มีฟิลด์ข้อมูลดังนี้:

| ฟิลด์ข้อมูล | ชนิดข้อมูล | คำอธิบาย | ตัวอย่างข้อมูล |
| :--- | :--- | :--- | :--- |
| `id` | Integer | ลำดับที่ของพัสดุ (1 - 49) | `1` |
| `name` | String | ชื่อพัสดุจัดซื้อทางการ | `"ซองใส่แผ่น CD ห่อ (แบบใส) 1 : 100 ซอง"` |
| `spec` | String | รายละเอียดคุณลักษณะเฉพาะพัสดุ | `"ซองพลาสติกใส ใช้สำหรับเก็บและป้องกันรอย..."` |
| `qty` | Integer | จำนวนที่จัดซื้อ | `400` |
| `unit` | String | หน่วยนับพัสดุ | `"ซอง"` |
| `unit_price` | Float | ราคาต่อหน่วยนับ (บาท) | `1` |
| `total_price`| Float | ราคารวมสุทธิ (จำนวน x ราคาต่อหน่วย) | `400.0` |
| `division` | String | กลุ่มงานที่เป็นผู้เบิกใช้พัสดุ | `"สถิติ"` |
| `image` | String | ชื่อไฟล์รูปภาพพัสดุจริง (ใน `public/รูปภาพ/`) | `"87203_0.jpg"` |

## 2. Category Classification Rules (กฎการจัดหมวดหมู่พัสดุแบบไดนามิก)
ระบบคำนวณแยกหมวดหมู่วัสดุอัตโนมัติ (Dynamic Classification) โดยพิจารณาคำสำคัญ (Keywords) ในฟิลด์ `name`:
* **connectivity (อุปกรณ์เชื่อมต่อ):** hdmi, type-c, usb hub, converter, console cable, sfp, adapter, สายแปลง, หัวสัญญาณ
* **storage (อุปกรณ์จัดเก็บ):** flash drive, sd card, hdd, ssd, จัดเก็บข้อมูล
* **peripherals (อุปกรณ์ต่อพ่วง):** mouse, keyboard, webcam, หูฟัง, กล้อง
* **electronics (อิเล็กทรอนิกส์):** raspberry, โมดูล, esp32, poe
* **tools (เครื่องมือช่าง):** บัดกรี, ลวดซับ, น้ำยาประสาน, ดีบุก, รางปีกนก, สต๊อปเปอร์, คลิป, din rail
* **organization (จัดระเบียบ):** กระเป๋า, เป้, ปลั๊ก, รางปลั๊ก, โรลม้วน
* **toner (หมึกพิมพ์):** หมึก, hp 230a, สีดำ, สีฟ้า, สีเหลือง, สีชมพู
* **consumables (อื่นๆ):** ถ่าน, ถ่านอัลคาไลน์
