## 1. UI Audit (การทบทวนปัญหาอินเทอร์เฟซผู้ใช้เดิม)
จากการประเมินโครงสร้างและภาพรวมอินเทอร์เฟซเดิม พบปัญหาเชิงการประยุกต์ใช้งานจริงดังนี้:
*   **Visual Hierarchy (ลำดับการจัดวาง):** หน้าแดชบอร์ดสรุปมีขนาดการ์ดที่เท่าๆ กัน ขาดการแยกความสำคัญระหว่างมูลค่าจัดซื้อและตัวแปรอื่นๆ
*   **Color System (ระบบสี):** การใช้สีพื้นเต็มบนการ์ดสถิติ (Full-colored Cards) ทำให้หน้าแดชบอร์ดมีสีฉูดฉาด ขาดความเป็นทางการระดับสถาบัน
*   **Typography (การเลือกใช้อักษร):** ข้อมูลตัวเลขอัตราเงินและสถิติใช้อักษรทั่วไป ซึ่งมีขนาดความกว้างไม่เท่ากัน (Proportional) ส่งผลให้การจัดตารางรายงานตัวเลขในคอลัมน์ไม่ตรงกัน
*   **Spacing & Grid Layout (ระยะช่องว่าง):** การจัดวางการ์ดพัสดุในหน้าตรวจรับค่อนข้างหนาแน่น ขาดพื้นที่ว่าง (White Space) เพื่อการลดความเหนื่อยล้าทางสายตาของคณะกรรมการ
*   **Mobile / Field Inspection (การใช้งานนอกสถานที่):** ปุ่มควบคุมและขนาด Touch Target บนหน้าตรวจรับพัสดุผ่านแท็บเล็ต/มือถือนอกสถานที่ค่อนข้างเล็กและกดยาก

## 2. Premium Design Direction (ทิศทางการออกแบบใหม่)
การปรับเปลี่ยนภาพลักษณ์ของระบบได้ยึดตามหลักการออกแบบสากลสำหรับองค์กรและสถาบันภาครัฐระดับสูง (Modern GovTech / Minimal Luxury):
*   **Calm, Precise and Authoritative (ความสงบ แม่นยำ และน่าเชื่อถือ):** ปรับลดความฉูดฉาดลง เน้นการออกแบบที่เป็นระเบียบ เรียบง่าย และดูเป็นทางการ
*   **Minimal Luxury & Executive Dashboard:** ใช้เทคนิคขอบสีทอง (Champagne Gold) และเส้นคั่นที่คมบาง ร่วมกับพื้นหลังสีอบอุ่น Off-white เพื่อความพรีเมียม
*   **Clean Data Presentation:** การจัดตารางและการ์ดพัสดุจะเน้นให้อ่านสเปกสินค้าและหมายเลขเครื่องได้ชัดเจนที่สุดโดยไม่มีสิ่งรบกวนสายตา (Clean Layout)

## 3. Design System & Tokens (ชุดการตั้งค่าโทนสีและไทโปกราฟี)
ระบบสไตล์ชีตได้รับการปรับโครงสร้างการอ้างอิงคลาส CSS ในระบบ **Tailwind CSS v4** ผ่านการกำหนด `@theme` ในไฟล์ `src/index.css` เพื่อหลีกเลี่ยงการ Hard-code ค่าสี:

### 3.1 Design Tokens (โทนสีสถาบันภาครัฐ)
```css
@theme {
  --color-gov-navy: #0b132b;          /* สีน้ำเงินเข้มสถาบันภาครัฐหลัก */
  --color-gov-blue: #1c2541;          /* สีน้ำเงินเข้มรองสำหรับเมนู/หัวเรื่อง */
  --color-gov-gold: #c5a880;          /* สีทอง Champagne Gold สำหรับขอบ/Badges */
  --color-gov-gold-light: #f9f6f0;    /* สีครีมทองอ่อนสำหรับพื้นหลังสถานะ/Active */
  
  --color-neutral-warm: #fcfbfa;      /* สีครีมขาว Off-white พื้นหลังระบบหลัก */
  --color-neutral-charcoal: #1e293b;  /* สีถ่านสำหรับฟอนต์หลัก เพื่อความคอนทราสต์ */
  
  --color-status-passed: #065f46;     /* สีเขียวมรกตตรวจผ่าน */
  --color-status-pending: #b45309;    /* สีส้มเหลืองรอการตรวจสอบ */
  --color-status-failed: #991b1b;     /* สีแดงเข้มตรวจไม่ผ่าน */
}
```

### 3.2 Typography & Tabular Figures
*   **Sans Font:** ผสานฟอนต์พรีเมียม **`Inter`** ร่วมกับฟอนต์ภาษาไทย **`Prompt`** เพื่อความสวยงามในระดับอินเทอร์เฟซผู้ใช้
*   **Serif Font (สำหรับรายงานราชการ):** ผสาน **`Sarabun`** เพื่อให้เอกสารตารางแนบท้ายเมื่อพิมพ์ออกทางเครื่องพิมพ์ (Print Layout) สอดคล้องตามระเบียบงานสารบรรณอย่างเป็นทางการ
*   **Tabular Numbers:** ใช้คลาส `.num-tabular` (เรียกคุณสมบัติ `tabular-nums`) บนตารางรายการ ราคาต่อหน่วย และตัวเลขแดชบอร์ด เพื่อควบคุมให้ความกว้างตัวเลขทุกตัวเท่ากัน ส่งผลให้การจัดแสดงงบประมาณและการเทียบยอดเรียงแถวตรงกันอย่างเป็นระเบียบ

## 4. Page-by-page Improvements (การปรับปรุงเชิงหน้าจอปฏิบัติงาน)

### 4.1 Dashboard (แผงควบคุมหลักสำหรับผู้บริหาร)
*   **Stat Cards:** ยกเลิกการใช้สีพื้นเต็มหน้าการ์ด เปลี่ยนเป็นใช้การ์ดขาวขอบเรียบคัทสติ๊กเกอร์บางๆ สีตามสถานะ (เช่น แถบทองสำหรับงบประมาณ, แถบน้ำเงินสำหรับพัสดุ, แถบเขียวสำหรับจำนวนที่ตรวจผ่าน)
*   **Progress Progress Bar:** ปรับแถบแสดงเปอร์เซ็นต์ความก้าวหน้าการตรวจรับรวมให้เป็นแถบเกรเดียนท์ไล่สีพรีเมียม (Midnight-to-Gold) ดูหรูหราและอ่านง่าย
*   **Committee Cards:** จัดวางการ์ดข้าราชการคณะกรรมการตรวจรับใหม่ให้มีเส้นขอบและป้ายประธาน/กรรมการสีทอง Champagne Gold ตัดกับพื้นขาวอย่างสง่างาม

### 4.2 Filters & Search Toolbar (แถบค้นหาอัจฉริยะ)
*   ปรับเปลี่ยนโครงสร้างให้กระชับ โค้งมน และใช้ขอบเส้นสีเทาอุ่นที่เบาบางลง
*   ช่องค้นหา (Search Box) ใช้ไอคอนแว่นขยายสีทอง Champagne Gold
*   ตัวกรองและช่องกรอกราคาต่ำสุด-สูงสุดจัดระเบียบให้มีความกว้างสม่ำเสมอ

### 4.3 Item Card (การ์ดแสดงรายการพัสดุรายชิ้น)
*   ติดตั้งแถบสีบ่งบอกสถานะการตรวจทางซ้ายของการ์ดแต่ละใบ ช่วยให้อ่านได้ทันที
*   มีระบบพรีวิวระบุ Badge ยี่ห้อหมวดหมู่ชัดเจน และไอคอน S/N หรือรูปถ่ายพัสดุที่จะแสดงผลเฉพาะรายการที่มีหลักฐานจริงเท่านั้น
*   ใช้เงาบางเบาสไตล์พรีเมียม (`shadow-premium`) และเมื่อเลื่อนเมาส์ผ่าน (Hover) จะเกิดการขยับขึ้นสุภาพ 0.5px พร้อมขอบขยายแสงสีทองบางๆ

### 4.4 Detail Modal (Inspection Workspace)
*   **Header:** ใช้หัวแถบน้ำเงินเข้มขอบล่างเส้นทอง Champagne Gold ระบุรหัส ID พัสดุชัดเจน
*   **Checklist Layout:** ติ๊กบล็อก Checklist ตรวจรับ 8 ข้อ ใช้รูปแบบสีมรกตอ่อนตัดกับขอบโค้งมนเพื่อระบุความผ่านเกณฑ์
*   **Timeline & Audit Logs:** ประวัติการแก้ไข (Audit Trail) จัดอยู่ในตารางการ์ดเล็กสีขาวแยกประเด็นวันเวลาและรายละเอียดชัดเจน

## 5. Responsive & Tablet Strategy (กลยุทธ์การแสดงผลตามขนาดหน้าจอ)
*   **Desktop View:** ระบบนำเสนอในรูปแบบ Sidebar แยกกลุ่มฟังก์ชัน เมนูแสดงสถานะพรีเมียม และ Layout Grid 3 คอลัมน์สำหรับแสดงพัสดุทั้งหมด
*   **Tablet View (สำหรับการเดินตรวจพัสดุหน้างาน):** ปรับ Grid เป็น 2 คอลัมน์ และขยาย Touch Target ขนาดของช่องปุ่ม Checklist ให้กว้างอย่างน้อย 44x44px เพื่อความสะดวกในการใช้นิ้วมือกดขณะเดินตรวจรับในสนาม
*   **Mobile View:** ซ่อนแท็บเมนูด้านซ้ายและเปลี่ยนเป็นแผงเมนูด้านบนที่กระชับ การ์ดพัสดุแสดงผล 1 คอลัมน์ชัดเจน และปุ่มตรวจรับ modal แสดงเต็มหน้าจอสมาร์ตโฟน (Full-screen Overlay) เพื่อลดการสั่นไหวและเลื่อนหลุดของนิ้วมือ

## 6. Accessibility & Compliance Guidelines
*   **Contrast ratio:** การใช้สีน้ำเงินเข้มและสีเขียวมรกตผ่านการทดสอบเรื่อง Contrast Ratio ทำให้อ่านข้อมูลสเปกพัสดุได้ง่ายแม้อยู่ในที่กลางแจ้งที่มีแดดจัด
*   **Keyboard Navigation:** รองรับการทำ Focus Ring สีทองเมื่อใช้แป้นพิมพ์สลับปุ่มเลือกเมนู
*   **Designed for Future Compliance:** โครงสร้างพร้อมรองรับการทำ Screen Reader Aria-labels และ Contrast switcher สำหรับหน่วยงานในขั้นตอนถัดไป

## 8. Recommendations & Implementation Priorities (ข้อเสนอแนะและลำดับความสำคัญ)

### 🟢 ดำเนินการเสร็จสมบูรณ์แล้ว (Implemented)
*   **Design Tokens & Themes:** บันทึกโครงสร้างสี Midnight Blue, Champagne Gold และ neutral warm ลงในสไตล์ชีต Tailwind v4 เรียบร้อยแล้ว
*   **Premium Component Refactor:** ปรับการ์ดพัสดุ, แดชบอร์ดสรุปสถิติกราฟ SVG, แผงควบคุมตัวกรอง และหน้าพิมพ์รายงาน PDF ให้สวยงามระดับองค์กร

### 🔵 แผนงานเพื่อรองรับในอนาคต (Planned / Future Enhancements)
*   **Critical:** พัฒนาระบบบีบอัดขนาดไฟล์ภาพถ่าย (Image Canvas Compression) เพื่อควบคุมพื้นที่แคช LocalStorage ไม่ให้เต็มขีดจำกัด 5MB เมื่อมีการแนบภาพหลักฐานพัสดุจำนวนมาก
*   **High:** การเพิ่มเมนูสวิตช์โหมด Contrast สูงสุด (High Contrast Switcher) สำหรับช่วยเหลือผู้บกพร่องทางสายตา
*   **Medium:** ระบบสร้างความเคลื่อนไหวจำลองโหลดข้อมูล (Skeleton Loading) ขณะนำเข้าไฟล์ Excel โครงการใหญ่
